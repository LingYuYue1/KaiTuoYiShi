# 开拓轶事 KaiTuoYiShi

开拓轶事是一个以 **React 19 + TypeScript + Vite + Tailwind CSS** 构建的同人叙事 Web 应用。它尝试把 AI 互动小说、跑团式剧情推进、世界书、结构化变量、伙伴档案、手机通讯、新闻演化、记忆压缩、原著资料智库和文生图资产管理整合成一个完整的前端游戏体验。

项目目前处于早期开发阶段，第一阶段聚焦“登上星穹列车”相关的开局流程与核心系统验证。它不是一个单纯的聊天壳，而是希望让 AI 生成的剧情真正落到可持续维护的游戏状态里：角色会留下档案，物品会进入背包，世界时间和地点会变化，伙伴会记住共同经历，新闻和手机会把故事延伸到主线之外。

> 本项目是非官方同人项目，仅用于学习、研究与非商业创作。

## 项目目标

开拓轶事的核心目标是做一个“可持续游玩”的 AI 叙事框架。

很多 AI 剧情应用能生成漂亮的段落，但难点在于长期游玩后容易失忆、人物重复、变量散乱、剧情推进不可控。这个项目试图用前端数据模型和多层提示词协议解决这些问题：

- 用世界书和提示词模块约束叙事风格、开局规则与变量边界。
- 用变量模型把正文中的事实结算为结构化存档。
- 用伙伴、背包、手机、新闻、记忆、智库等系统承接正文里发生过的事。
- 用可视化面板让玩家能检查、修正和继续扩展世界状态。
- 用文生图工作台逐步把角色头像、立绘、场景壁纸和相册资源接入叙事体验。

## 当前功能

### 新游戏与主线开局

- 五步“踏上旅途”向导：世界、角色、命途、史官、总览。
- 支持旅人姓名、性别、来历、外貌、性格、背景、能力等基础档案。
- 支持剧情模式、难度、起始场景、阵营与命途选择。
- 第一阶段默认围绕黑塔空间站与星穹列车相关开局展开。

### AI 主剧情

- 支持流式 / 非流式回复。
- 主剧情输出分为正文、行动选项、记忆、动态世界等层次。
- 支持重新开局、重 roll、行动选项快捷输入。
- 回合正文会进入变量结算、记忆整理、新闻推演等后续流程。

### 世界书与提示词模块

- 内置开局规范、伙伴档案、变量系统、新闻、手机等提示词模块。
- 世界书管理支持内置与额外世界书分栏。
- 内置世界书可编辑、启用和禁用，便于玩家调试自己的叙事规则。
- 开局世界书预设会在首次启动时注入 IndexedDB。

### 变量系统

- 独立变量模型读取主剧情正文，并输出结构化变量命令。
- 支持更新旅人、世界、NPC、手机、背包、命途进度等状态。
- 对背包物品、NPC 档案、NSFW 档案、命途状态机等字段有校验和兜底。
- 会过滤错误字段、占位符对象、旧字段名和不该写入的系统。
- NPC 归一化层会合并同一人的别名、状态称呼和临时称呼，减少重复建档。

### 伙伴系统

- 伙伴 / 路人分栏管理。
- 记录姓名、别名、性别、关系、好感度、阵营、是否同行、初见与最近回合。
- 支持人物介绍、外貌、穿着、说话方式、性格、装备摘要和同行记忆。
- 原著角色会作为重要伙伴处理。
- 支持头像槽位、角色立绘、图像档案和 NSFW 档案预留。

### 背包与装备

- 方格背包 UI，支持分类筛选和物品详情浮层。
- 物品支持食物、消耗品、光锥、武器、衣服、饰品、纪念品、关键道具等分类。
- 装备系统包含光锥、武器、帽子、衣服、裤子、鞋子、饰品 I、饰品 II 八个槽位。
- AI 写入物品时有解析兜底，避免不完整或错误格式导致界面崩溃。

### 记忆系统

- 当前设计只保留即时、短期、长期三层记忆。
- 即时记忆用于记录每回合刚发生的原始事件。
- 短期记忆由即时记忆压缩而来。
- 长期记忆只保留稳定事实、主线转折、关系变化、不可逆后果和长期目标。
- NPC 同行记忆直接挂在伙伴档案中，服务角色关系连续性。

### 手机系统

- 手机作为独立通讯终端，不直接复用主聊天。
- 支持联系人、会话、主动来信种子、未读状态等数据结构。
- 手机内容可以读取主剧情记忆、NPC 档案、新闻和上下文。
- 后续目标是让短信、群聊和远程回应成为主线之外的叙事延伸。

### 新闻、剧情编织与智库

- 新闻系统用于后台事件和星际和平周报式内容推演。
- 剧情编织系统用于管理原著剧情轨道、自制剧情轨道和当前推进段。
- 智库系统用于保存和检索原著资料、角色、地点、阵营、术语、事件等。
- 内置公开预设资源位于 `public/zhiku-presets/` 和 `public/worldbook-presets/`。

### 相册与文生图工作台

- 相册以角色为中心管理视觉资源。
- 支持手动生成、资源库、角色锚点、场景壁纸、规则中心、生成队列、历史与整理等工作台入口。
- NPC 图像档案支持头像槽位、立绘、角色锚点和 NSFW 部位图槽位。
- 规则中心支持规则模板、模型规则集和风格预设。

## 技术栈

- React 19
- TypeScript
- Vite
- Tailwind CSS 3
- IndexedDB 本地存档
- OpenAI 兼容 / Claude / Gemini 风格的文本接口配置
- 纯前端状态管理与服务层拆分

## 本地运行

安装依赖：

```bash
npm install
```

启动开发服务：

```bash
npm run dev
```

构建生产版本：

```bash
npm run build
```

项目不会自带任何 API Key。运行后需要在应用设置中配置自己的模型接口。

## 项目结构

```text
components/  前端组件、布局和各游戏系统面板
data/        内置菜单、世界书、提示词、原著资料和系统预设
hooks/       顶层游戏状态、主剧情流程、记忆、新闻和存档工作流
models/      TypeScript 数据模型与归一化逻辑
prompts/     CoT 与系统提示词模块
public/      随包发布的公开世界书和智库预设资源
services/    AI 请求、存档、剧情推进、记忆压缩、检索等服务层
styles/      Tailwind、主题变量和全局样式
utils/       变量执行、路径解析、背包、相册、世界书等工具函数
```

## 开发状态

这个项目仍在快速迭代中，当前代码更接近“可运行的原型 / 开发中版本”，而不是稳定发行版。

已经重点打磨的方向：

- 主界面三栏布局与系统抽屉。
- 新游戏向导。
- API 设置面板。
- 世界书管理。
- 变量系统与变量管理。
- 伙伴、背包、装备、记忆、新闻、剧情编织、智库、相册等主要系统骨架。

仍在继续完善的方向：

- 更稳定的剧情推进与异常恢复。
- 更完整的手机聊天工作流。
- 更细的剧情编织冲突处理。
- 文生图任务的自动化和队列管理。
- 大型前端包体拆分与性能优化。
- 更完善的测试覆盖。
