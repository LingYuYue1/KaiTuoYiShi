# 更新报告

## v0.2.1 测试版 - 2026-05-28

本次测试版重点继续修复记忆、变量写入和手机主动来信链路，为后续剧情编织优化做准备。

### 今日改动

- 补充“开拓轶事版 NPC 记忆写入法则”，让变量模型在每次有效互动后审计 NPC 的同行记忆、关系、好感、称呼、同行状态和稳定档案字段。
- 强化 NPC 入档规则：未建档角色需先判定重要性并查重；新入档时补写关键前情，避免记忆从当前回合突然断层。
- 收束 NPC 字段写入范围，避免模型写出不存在的 `age / faction / location / injury / present` 等字段；没有独立 NPC 字段的状态改写入 `介绍 / 同行记忆 / 世界事件`。
- 强化 NPC 记忆压缩提示词：压缩结果改为单段自然中文，限制在该 NPC 自身可知范围内，保留稳定关系、称呼、承诺、亏欠和冲突原因。
- 修正 NPC 记忆压缩提示词旧默认迁移，并更新提示词污染清洗规则。
- 修复手机主动来信不够“主动”的问题：主剧情产生来信种子后，玩家打开手机会自动生成一条最高优先级真实短信。
- 手机来信种子现在尊重手机系统设置：系统关闭、主动来信关闭或每回合上限为 0 时不再写入种子；超过上限会进入变量警告。

### 明日计划

1. 剧情编织优化
   - 降低剧情编织对主剧情的强制影响，避免滑窗内容在玩家尚未推进过去时反向绑架当前正文。
   - 区分“可参考剧情素材”和“当前必须承接剧情”，只把当前段落目标、人物关系和未结事项作为软锚点。
   - 增加剧情编织注入门槛：未满足玩家行动、地点、人物或章节条件时，只提示背景气氛，不强迫推进。
   - 检查设置页上下文预览与真实主剧情请求是否一致。

2. 手机端适配收尾
   - 继续处理剩余手机端拥挤、遮挡、滚动和按钮换行问题。
   - 保持桌面端布局不动，所有手机修复优先用移动端样式与断点恢复。
   - 重点复查系统面板、设置页、剧情编织和手机终端在窄屏下的可用性。

3. 积存 bug 清理
   - 汇总前面反复提到但尚未完成的积存问题，按“能复现 / 影响主流程 / 变量写入风险 / UI 阻塞”排序。
   - 优先解决会导致主剧情失忆、变量写入失败、手机/伙伴系统断链的 bug。
   - 新增忆庭 / reroll 专项：检查 reroll 后回合纪要编号跨越过大、摘要索引跳号，以及 reroll 后内容有时与 reroll 前一致的问题。
   - 重点排查 reroll 是否正确回滚并重写忆庭纪要、即时记忆、变量批次、随机性参数和主剧情请求上下文，避免旧摘要残留或复用旧回复。
   - 每个修复后至少跑 `npx.cmd tsc --noEmit`，涉及前端体验或发布前再跑 `npm.cmd run build`。

### 验证

- `npx.cmd tsc --noEmit` 待本批文档与版本号修改后执行。
- `npm.cmd run build` 待发布前执行。

## v0.2.0 - 2026-05-28

本次更新重点完成手机端体验整理，并修复若干会影响叙事连续性的关键问题。

### 新增

- 开局选择命途时，现在可以选择初始命途阶段。
- 可选阶段包括：浅涉、践行、深诣、伪令使、令使。
- 所选阶段会写入旅人命途档案，并进入开局全局事件、开局档案和总览确认页。

### 手机端适配

- 优化手机底部快捷菜单，减少系统面板和底部栏互相遮挡。
- 调整背包页手机布局，物品详情现在位于下方独立区域，不再与物品列表重叠。
- 优化剧情编织、智库、命途、装备、战技、存档、设置等面板在窄屏下的滚动、字号、按钮排列和内容密度。
- 保持桌面端布局习惯，手机端改动尽量通过响应式样式单独生效。

### 修复

- 暂停生成后，本轮已提交文本会回到输入框，方便玩家修改错字后重新发送。
- 终止生成时会回滚本轮临时消息，不再保存半截回复或错误推进回合。
- 纯拟声词、动物叫声和场景声音不再默认当作玩家说话。
- 手机回复现在会读取最近主剧情回顾和当前手机会话摘要，减少手机聊天与正文上下文脱节。
- 修复伙伴同行记忆压缩时误把压缩提示词写进 NPC 记忆的问题；历史污染内容在注入时也会被清洗。

### 验证

- `npx.cmd tsc --noEmit` 通过。
- `npm.cmd run build` 通过。
