# 更新公告

## v0.3 测试版 - 2026-05-29

这次更新合并了昨晚 `v0.2.1` 的收尾内容、今天下午到凌晨的剧情编织重做，以及一整轮玩家反馈的关键 bug 修复。  
简单说，这是一次偏“地基重修”的版本：主剧情不再被剧情编织强行拖着走，记忆、变量、手机、存档、reroll、NPC 档案这些会影响长期体验的系统都做了加固。

这批改动跨度很大，也确实花了很久：从昨天下午到今天凌晨，我们围绕“主剧情连续性”和“长期游玩稳定性”连续排查、重构、补测试，目标是让后续测试不再被同一类失忆、串档、重复剧情、变量污染问题反复打断。

### 一句话总结

- 剧情编织从“强制玩家走主线”改成“原著剧情参考与章节锚点系统”。
- 主剧情优先承接玩家输入、已发生事实、即时剧情回顾、剧情回忆和 NPC 记忆。
- 已归档剧情不会再轻易回流成当前事件，避免打过的 boss、完成的桥段被重新演一遍。
- reroll、存档、手机、记忆、变量等运行时副作用补上回滚和隔离。
- 变量系统进一步减少对玩家档案的干扰，并增强 NPC、手机、NSFW 档案等事实落库能力。
- 手机端 UI 继续补齐，记忆系统等面板在窄屏下不再挤成一团。

## 重点更新：剧情编织系统重做

### 原著剧情从“强推主线”改为“参考辅助”

旧版剧情编织最大的问题，是它容易把原著剧情当成当前回合必须发生的内容。玩家明明已经用自己的方式解决了某个桥段，后续主剧情到了原著对应位置时，仍可能再次把旧事件拉回来。

本版调整后：

- 原著剧情主要作为参考素材和章节方向，不再直接覆盖玩家当前行动。
- 玩家已经完成的事件会被视作历史事实，而不是等待复演的原著节点。
- 当前回合会优先承接玩家正在做的事，再参考剧情编织给出的章节目标、人物关系、未结问题。
- 剧情编织注入会区分“当前段”“后续段”“已归档段”，避免把未来或过去的段落混进当前正文。

### 引入已分解剧情预设

我们替换并接入了已分解的内置剧情编织预设。  
这次测试也确认了一个关键结论：之前剧情不推进、结束判定困难，很大程度上来自原始剧情素材没有按段拆好。

现在的内置预设具备：

- 系列、章节、分段结构。
- 当前段目标。
- 结束条件。
- 关键角色。
- 关键地点。
- 可承接的历史事实。
- 后续段落提示。

这让剧情编织不再只是一大段原著文本，而是更适合游戏运行时判断的章节材料。

### 自动推进逻辑重做

剧情编织推进不再只靠“当前回合写到了某个关键词”粗暴判断，而是加入了更清楚的阶段判断：

- 当前分段是否已经出现明确收束。
- 玩家行动是否真的推进了该段，而不是只是提到相关词。
- 后续分段是否有足够命中。
- 当前段是否已归档。
- 旧锚点是否需要自愈。
- 门禁结果是否允许写回锚点。

如果暂时不推进，面板会显示最近判定理由，方便继续调试。

### 已归档剧情防重演

这次重点修了“已经发生的原著段落被重新拿出来演”的问题。  
例如“某个 boss 已经被击退”这类内容，现在会作为历史事实承接，而不是再次生成一只同样的 boss 让玩家重复打。

机制上做了几层保护：

- 已归档段不会再作为当前注入段。
- 当前素材块会切到后续段。
- 旧段概括不会残留在当前块里。
- 可以保留“已经完成”的历史事实，用于 NPC、新闻、手机、主剧情承接。
- 状态自愈只修正锚点，不会伪造新的剧情进展记忆。

### 剧情编织与其他系统联动

这次没有只改剧情编织本体，而是把周边系统也接到同一套章节锚点上，避免各系统各讲各的。

- 主剧情：读取当前进度、历史归档、当前段软目标，但不得被原著强制绑架。
- 新闻系统：只承接已发生的公共事实和余波，不提前宣布未来剧情。
- 手机系统：可以根据已发生事件触发合理来信，但不能替玩家推进章节。
- NPC 记忆：剧情段落完成后，会给相关且有承接价值的 NPC 写入同行记忆。
- 即时记忆 / 忆庭：章节进度会沉淀为可召回事实，避免后续只靠聊天滑窗记住。
- 调试上下文：每回合请求上下文会记录剧情编织诊断，方便确认模型到底看到了什么。

### 剧情规划与关系规划

本版加入了更像“调度快照”的规划分析：

- 剧情规划关注当前章节、锚点、已完成段、待解问题和后续建议。
- NPC 关系规划不再绑定“女主路线”，而是面向所有重要 NPC。
- 关系规划依据共同经历、承诺、冲突、信任、称呼和同行记忆。
- 关系规划从常驻信息块移到独立 tab，避免把同行记忆全文塞进右侧栏。

这部分的目标不是替玩家决定路线，而是帮助系统理解“哪些关系值得继续承接”。

## 重点修复：reroll 与回合副作用

### 修复 reroll 后正文不变的问题

玩家反馈：重 roll 后，新正文和旧正文一模一样。  
进一步排查发现，不只是正文没有变化，变量、记忆等副作用也没有正确回退。

本版修复：

- reroll 前保存完整深拷贝快照。
- reroll 时恢复聊天、旅人、世界、记忆、忆庭、智库、手机、NPC、相册、新闻、剧情、剧情编织、变量批次、队列任务和回合数。
- reroll 请求增加随机扰动，避免直接复用旧回复。
- 旧后台流程如果在 reroll 后继续返回，会被工作流版本号挡掉，不能再把旧副作用写回状态。

### 修复 reroll 后忆庭编号跳号

玩家反馈：reroll 后忆庭回合编号跳号，被 reroll 掉的即时记忆仍被保存。  
本版把 reroll 和后台结算链路做了统一保护：

- 记忆压缩写入前检查当前工作流是否仍有效。
- 忆庭入库前检查当前工作流是否仍有效。
- 新闻、变量、剧情编织推进、NPC 记忆压缩、自动存档等后台结算全部加上过期保护。
- turnCount 持久化也必须确认工作流仍有效。

现在 reroll 不再只是“换正文”，而是会把该回合相关副作用一起回退。

## 重点修复：存档隔离与手机串档

### 修复多个存档内容互串

玩家反馈：A 存档的信息会出现在 B 存档里。  
排查发现，手机系统曾经存在全局备份，会在读档时和当前存档合并，导致通讯录、聊天记录、主动来信种子跨档污染。

本版修复：

- 手机通讯录、聊天、主动来信种子只属于当前存档。
- 读档时手机状态只来自 `save.手机`。
- 不再用全局 `phoneSystemState` 合并当前存档。
- reroll / abort 回滚时也不再写全局手机备份。
- 存档归一化只修当前存档本身，不拉入其他档案的运行时数据。

现在手机聊天记录和通讯录不会再跨存档互串。

### 明确运行时事实不能写入全局 setting

本次也统一了边界：

- 记忆。
- 忆庭。
- NPC。
- 新闻。
- 手机通讯录与聊天。
- 主动来信种子。
- 变量批次。
- 当前剧情进度。

这些都属于单个存档的运行时事实，不应进入全局设置。

## 重点修复：变量系统与玩家档案保护

### 玩家手写档案不再被变量系统覆盖

玩家反馈：自己认真写的外貌、性格、身份、背景会被变量系统改掉。  
本版明确保护玩家核心档案：

- `旅人.身份`
- `旅人.外貌`
- `旅人.性格`
- `旅人.背景`
- `旅人.能力`
- `旅人.专长知识`
- `旅人.穿着`

变量模型即使输出旧命令，也会在进入执行链路前被静默过滤。  
事实协议里的 `traveler_profile` 也不再作为红色警告刷屏，而是作为已忽略记录处理。

这能同时做到两点：

- 玩家手写内容不会被 AI 自动替换。
- 变量系统少处理一类高风险、低收益内容，减少噪音。

### 变量事实协议继续加固

变量系统仍保持“主剧情给事实草稿，变量模型提取事实，前端确定性转换命令”的方向。

本版继续加固：

- NPC 事实支持稳定 id 和中文姓名互查。
- NPC 记忆支持更稳的摘要写入。
- 手机来信种子能补全目标 NPC id。
- 时间推进继续遵守当前时间锚点，不把时间往回写。
- NSFW 档案新增独立事实类型。
- 旧 `<变量更新>` 仍保留兼容，但不再是主路径。

## 重点修复：NPC 记忆与角色连续性

### 强化 NPC 记忆写入法则

昨晚 `v0.2.1` 中补上了“开拓轶事版 NPC 记忆写入法则”，这次合并进 `v0.3`。

现在变量模型在每次有效互动后，需要审计：

- 是否要追加同行记忆。
- 是否要刷新关系。
- 是否要调整好感。
- 是否要记录称呼变化。
- 是否要记录同行状态。
- 是否要更新稳定档案字段。

这能减少“刚互动过几回合，NPC 就像没见过玩家”的情况。

### NPC 入档和查重更稳

- 未建档角色会先判断是否重要。
- 新入档时补写关键前情。
- 原著稳定 id 支持自动补档。
- 原创 NPC 不会因为匹配不到原著 canonical 而错误合并。
- 有记忆、好感变化、同行关系的重要 NPC 会倾向进入 companion 层，而不是很快掉成路人。

### NPC 记忆压缩提示词修复

修复并清洗了 NPC 同行记忆压缩提示词污染：

- 压缩结果必须是 NPC 自身可知范围内的自然中文。
- 保留稳定关系、称呼、承诺、亏欠、冲突原因。
- 不把压缩说明本身写进 NPC 记忆。
- 历史污染内容在注入主剧情时也会被清洗。

## 重点修复：手机系统

### 手机聊天会反哺主剧情

玩家反馈：第 20 回合和 NPC 在手机里聊天，第 21 回合主剧情里 NPC 不知道聊过什么。  
本版修复：

- 手机 AI 回复后，会立即写入一条即时记忆。
- 私聊对象绑定 NPC 时，会同步写入该 NPC 的同行记忆。
- 记忆来源标记为手机。
- 手机本地摘要仍保留，但不再是主剧情能知道手机内容的唯一入口。

现在手机聊天不是孤立小系统，而会成为后续主剧情可承接的事实。

### 主动来信不再长期沉默

玩家反馈：手机几乎不会触发 NPC 主动来信。  
本版修复：

- 变量模型会更积极审计低频 `phone_seed`。
- 主剧情回合结束后有低频兜底来信种子生成。
- 打开手机时会自动处理最高优先级 pending seed，生成真实短信。
- 来信种子尊重手机系统设置和每回合上限。
- 新增 `phone` 后台队列任务，能在变量面板看到手机来信任务。

主动来信不会变成每回合刷屏，但也不应该再长期完全不触发。

## 重点修复：正文归属、拟声词与“玩家夺舍 NPC”

这是一个反复修了多次的顽固问题：  
环境音、拟声词、生物吼叫、旁白短句，有时会被挂到玩家头像下面；NPC 的话偶尔也会被误判成玩家说话。

本版做了两层处理：

### 生成前约束

- 提示词继续要求旁白、环境音、动物叫声、机械声不能写成玩家发言。
- 玩家不能替 NPC 说话。
- NPC 台词必须归属到对应 NPC。

### 保存与渲染层清洗

仅靠提示词不够，所以本版也改了保存和渲染层：

- 增强 `playerSpeechGuard`。
- 增加“隆隆”“轰隆隆”等拟声识别。
- 旁白整句引号不再自动提升为玩家气泡。
- 只有玩家输入存在明确说话证据时，才允许把某些内容渲染成玩家发言。
- 新正文保存前会清洗错误归属。
- 清洗后的正文会进入后续记忆、变量、新闻、忆庭，避免错误归属继续污染系统。

这不是只改显示美化，而是尽量从正文保存链路上减少错误归属被长期记住。

## 重点修复：NSFW 档案

玩家反馈：开启 NSFW 档案后也不会更新。  
本版新增 `nsfw_archive` 变量事实协议，让 NSFW 档案不再只依赖旧变量命令猜路径。

新增能力：

- 支持 intimacyStage、boundaries、preferences、sensitivePoints、taboos、experiences、longTermFacts 等字段。
- 支持女性身体档案字段。
- 支持男性身体档案字段。
- 只更新已入档 NPC。
- 旧命令写 NSFW 档案时也会经过策略检查。

同时加入禁写边界：

- 帕姆。
- 佩佩。
- 怪物。
- 裂界生物。
- 反物质军团怪物。
- 机兵。
- 机器人。
- 人偶。
- 投影。
- 生物形态或非人对象。

这些对象暂时不进入 NSFW 档案系统。

## 新增：开局随机生成人设模板

玩家反馈：不会写外貌、性格、背景，希望有模板。  
本版在玩家信息页加入随机生成角色模板功能：

- 使用主 API 模型生成。
- 可生成外貌、性格、背景等开局档案。
- 玩家不满意可以再次生成。
- 旁边新增提示词输入框，玩家可以写偏好，例如“公司背景”“冷静理性”“战斗型”“黑塔空间站相关”等。
- 按钮旁增加小字提示：该功能走主 API 模型。
- 提示词要求生成的是玩家角色模板，不是弱路人 NPC 模板。
- 不默认让玩家取代星 / 穹，也不默认已经是列车组成员。
